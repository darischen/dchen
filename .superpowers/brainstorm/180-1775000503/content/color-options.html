Color Palette Options: Better ML/Web Differentiation

Three ways to make ML and Web projects visually distinct

Option A: Cyan + Green
ML/AI
#00d9ff
Web
#10b981
Hardware
#7c3aed

Cool cyan, warm green, deep violet. Natural contrast, easy to distinguish.

Cyan + Emerald Green

Vibe: ML stays cutting-edge (cyan), Web gets growth/balance (green), Hardware stays technical (violet).

Pro: Highly distinct. Green is familiar in tech (growth, health). Good variety.

Con: Less monolithic "cool technical" feel. Green might feel less serious than cyan/violet.

Option B: Bright Cyan + Slate Blue
ML/AI
#00f5ff
Web
#4f46e5
Hardware
#7c3aed

Bright cyan for ML (cutting-edge), indigo for Web (modern), violet for Hardware (technical).

Bright Cyan + Indigo

Vibe: All cool tech spectrum. ML is brightest (most innovative), Web is mid-tone (solid), Hardware is deepest (most technical).

Pro: Cohesive cool aesthetic. Gradient of brightness = hierarchy of complexity?

Con: Still fairly similar family. Might need saturation to distinguish.

Option C: Cyan + Pink
ML/AI
#00d9ff
Web
#ec4899

Cyan for ML (cool), pink for Web (warm contrast), violet for Hardware (technical).

Cyan + Hot Pink

Vibe: Bold contrast. ML stays cool/technical, Web gets energy/dynamism, Hardware stays grounded.

Pro: Maximum visual distinction. Pink feels modern but might read less "professional."

Con: More personality = less "serious mid-tier SWE." Breaks the cool-only aesthetic.

Which direction works better?

A (Green): If you want maximum clarity and don't mind the warmth of green.

B (Indigo): If you want all cool tones with a brightness gradient to distinguish them.

C (Pink): If you want bold contrast and aren't worried about the vibe shift.
